# TickDay Widget 显示逻辑需求文档

## 1. 当前问题

- Widget 固定显示用户选择的一个倒计时，到期后无变化
- 已过期事件若不被手动删除，始终占据 Widget
- Widget 不会自动切换到下一个有意义的事件

## 2. 设计目标

**Widget 始终自动展示当前最有意义的倒计时，零用户干预。**

## 3. 显示优先级

```
有未到期倒计时 → 显示最近未来事件（最紧迫）
       ↓ 无
有刚过期的倒计时（≤3天） → 显示最近过期事件（缓冲期）
       ↓ 无
全部已过期超过 3 天 → 显示空状态引导
       ↓ 无
列表完全为空 → 显示占位引导
```

### 3.1 有未到期倒计时

- 从所有未到期事件中，取「目标日期 - 今天」最小正值（最近将来）
- 如有多个同天事件，按创建时间排序取最早那个
- **正计时事件**（纪念日/已过去）永远有效，参与正常排序

### 3.2 全部已过期

| 过期时间 | 处理 |
|---|---|
| ≤ 3 天 | 继续显示，天数显示为「+1天」「+2天」「+3天」，让用户感知刚过去 |
| > 3 天 | 自动跳过，不显示在 Widget 中 |

如果列表中存在正计时事件，正计时事件视为「未到期」，按 3.1 规则处理。

### 3.3 列表为空

显示占位引导：
- 主文案：「添加一个倒数日」
- 副文案：「点击进入 App」
- 视觉：空心圆环 + 加号图标

## 4. 刷新时机

| 触发条件 | 行为 |
|---|---|
| iOS Widget Timeline 刷新（系统触发） | 重新排序，取当前应显示的事件 |
| 日期变更（跨天） | Timeline entry 设置 `Calendar.current.startOfDay(for: tomorrow)` 作为刷新节点 |
| 用户在 App 内增/删/改事件 | 调用 `WidgetCenter.shared.reloadAllTimelines()` 主动刷新 |

## 5. 不做的功能

- **置顶/固定事件**：增加复杂度，违背极简定位。如果用户最关心某事件，把日期设对即可。
- **手动选择显示哪个**：当前已有此功能，保留但降级为可选入口。默认行为改为自动。
- **Widget 内交互**：iOS Widget 交互能力有限，坚持纯展示，点击进 App。

## 6. 数据结构兼容

当前数据模型预计已包含：

```
Event {
    id: UUID
    title: String
    targetDate: Date
    eventType: .countdown | .anniversary    // .countdown=倒计时, .anniversary=正计时
    createdAt: Date
    category: String?
}
```

关键计算字段：

```
remainingDays = Calendar.current.dateComponents([.day], from: today, to: targetDate).day

// remainingDays > 0  → 未来，未到期
// remainingDays == 0 → 今天
// remainingDays < 0  → 已过去 |remainingDays| 天
```

## 7. 验收用例

| # | 场景 | 预期 Widget 显示 |
|---|---|---|
| 1 | 3个未到期事件（距今天 5天、10天、30天） | 显示「5天」那个 |
| 2 | 全部已过期，最近一个过期 1 天 | 显示「+1天」那个 |
| 3 | 全部已过期超过 3 天 | 显示空状态引导 |
| 4 | 今天新增一个事件后切回桌面 | Widget 立即刷新显示新事件 |
| 5 | 跨天后 | Widget 自动更新天数和排序 |
| 6 | 1个纪念日（已过去100天）+ 1个未到期（30天后） | 显示「30天」那个（未来优先） |
| 7 | 只有纪念日事件 | 正常显示该纪念日正计时 |
